# camera_bot

Telegram-бот для эксплуатации парка IP-камер по ONVIF: инвентаризация, диагностика, снимки, фоновый мониторинг доступности, аналитика отказов, интеграция с коммутаторами (Cross-24/KT-NOS, Huawei VRP) и Google Sheets/Drive.

Чистый Python + `requests` (long-polling, без сторонних Telegram-фреймворков — устойчив к дёрганому каналу с DPI). **164 команды**, 44 модуля, фоновый health-check, предиктивная аналитика, отчётность.

> ⚠️ Репозиторий содержит **только код**. Ни токенов, ни сервис-аккаунтов, ни данных инвентаря здесь нет — всё это задаётся локально в `tg_bot_config.json` (см. установку) и никогда не коммитится (`.gitignore`).

## Возможности (кратко)

- **Камеры:** `/shot`, `/diag`, `/info`, `/rtsp`, `/find` (скан подсети), `/clip` (видео, нужен ffmpeg).
- **Инвентарь:** поиск `/cam` по имени/IP/MAC/серийнику, `/where`, `/port`, `/export`, `/audit`, качество данных `/dq` `/lint`.
- **Надзор за парком:** фоновые TCP-пробы всех камер, алерты «упала/поднялась», `/report`, `/offline`, `/uptime`, `/top_flaky`.
- **Аналитика:** детект залипших кадров `/imgqa`, дрейф часов `/clock_report`, битрейт RTSP `/bitrate`, прогноз отказов `/risk`, `/sla`, теплокарты.
- **Сеть/свитчи:** топология `/topo`, PoE `/poe`, бэкап конфигов, детект чужих MAC, `/netcheck`.
- **Google:** дифф-синк инвентаря в таблицу `/sync`, снимки в Drive, дашборд, ночные задачи.
- **Операции (с подтверждением):** PoE-ребут `/reboot`, пусконаладка заводской камеры `/provision`.
- **Эксплуатация:** зоны, обходы `/patrol`, тикеты, QR-наклейки, смены, ППР, гарантии.
- **Самодиагностика бота:** `/status`, `/debug` (диаг-архив), `/upgrade` (обновление из git), `/log`, метрики.

Полный справочник по каждой команде — [`docs/bot_command_reference.md`](docs/bot_command_reference.md).

## Требования

- Windows 10/11 (использует Task Scheduler для автозапуска; ядро бота кроссплатформенное, но часть сетевых утилит — под Windows).
- Python 3.10+ (разрабатывалось на 3.14).
- Telegram-бот от [@BotFather](https://t.me/BotFather).
- Опционально: Google-сервис-аккаунт (для `/sync`, Drive, Calendar), доступ по SSH/web к коммутаторам (для функций сети).

## Установка

```powershell
git clone https://github.com/realgank/camera-bot.git
cd camera-bot
powershell -ExecutionPolicy Bypass -File .\install.ps1
```

Установщик поставит зависимости, создаст `tg_bot_config.json` из шаблона и проверит компиляцию. Затем:

1. Впишите токен от @BotFather в `token` в `tg_bot_config.json`.
2. Запустите: `.\run_bot.cmd` (или с автозапуском — `install.ps1 -RegisterAutostart`).
3. Напишите боту в Telegram — **первый написавший становится владельцем** (TOFU). Дальше `/help`.

Google-функции подключаются позже: положите `service-account.json` рядом и задайте `sheet_id`/`drive_folder_id` в конфиге.

## Конфигурация

Все настройки — в `tg_bot_config.json` (шаблон: [`config/tg_bot_config.example.json`](config/tg_bot_config.example.json)). Отсутствующие ключи берут разумные значения по умолчанию из `bot_state.py` — минимально нужен только `token`. Подсети сканирования, креды камер, пороги мониторинга, интервалы — всё переопределяется в конфиге.

Токен можно передать и через переменную окружения `TG_BOT_TOKEN` (приоритет над файлом).

## Безопасность

- Токен, сервис-аккаунт и данные инвентаря **не входят** в репозиторий и закрыты `.gitignore`.
- Бот служит только владельцу (TOFU + проверка `from.id`), с rate-limit.
- Операции, меняющие инфраструктуру (PoE, смена IP камеры, правки конфигов свитчей, запись в инвентарь), — только через двухшаговое подтверждение.
- Пароли камер бот **никогда не меняет**.
